# ホームページ管理ログ

## 2026年5月10日

### 1. プロジェクトの開始とフォルダ作成
- **目的**: 新しい会社のホームページを作成する。
- **プロジェクトフォルダ**: `G:\プロジェクト\architect` を作成。

### 2. CLI管理への要望
- WordPressをできるだけ触らず、Gemini CLIだけでホームページを編集・更新したい。
- 簡単に編集・更新できるようにしたい。

### 3. 技術選定の決定
- **アプローチ**: CLIでの管理のしやすさ、簡単な更新を考慮し、「静的サイト」を構築するアプローチを採用。
- **理由**: WordPressは管理画面が前提のため、CLI管理には不向き。静的サイトはHTML/CSS/JavaScriptファイルを直接CLIで編集できるため適している。
- **(オプション)**: コンテンツ更新の簡易化のため、Markdownと静的サイトジェネレーターの活用も検討。

### 4. 既存WordPressサイトへの影響
- 新しい静的サイトは、既存のWordPressサイトとは**完全に独立**して作成・運用される。
- 既存WordPressサイトの内容が変わったり、壊れたりすることは**ない**。

### 5. サーバーとドメインの運用
- **サーバー**: 新しい静的サイトの公開には、別途サーバー（ホスティングサービス）が必要になる（無料または安価なオプションが豊富）。
- **ドメイン**: 既存のドメイン（例: `yourcompany.com`）を新しい静的サイトで使用することは可能。
- **方法**: 新しい静的サイトは**サブドメイン**（例: `new.yourcompany.com`）を設定して運用する。
- **DNS設定**: ドメインのDNS設定を一部変更する必要がある（ホスティングサービスの手順に従う）。

## 2026年5月11日 (GitHub Pagesでの公開と独自ドメイン設定)

### 1. GitHub Pagesでのホームページ公開準備
- **課題**: GitHub Pagesが任意のサブディレクトリからの公開を直接サポートしていない（ルートまたは`/docs`のみ）。
- **解決策**:
    - ローカルの`G:\プロジェクト\architect`にあったホームページファイルを、`G:\docs`ディレクトリに移動。
    - `G:\プロジェクト\architect`ディレクトリを削除。
    - これらの変更をGitでステージ、コミット、プッシュ。
- **結果**: GitHub Pagesの設定で`/docs`ディレクトリを選択できるようになり、`https://yukiko-archi.github.io/yukikos-Obsidian-File/`で公開を確認。

### 2. 独自サブドメイン（archi.chitose-d.net）の設定
- **目的**: 公開されたホームページに独自サブドメイン`archi.chitose-d.net`を適用する。
- **手順**:
    - `G:\docs`ディレクトリ内に`CNAME`ファイルを作成し、内容を`archi.chitose-d.net`として保存。
    - `CNAME`ファイルをGitでステージ、コミット、プッシュ。
    - GitHub Pagesの設定画面で、カスタムドメインとして`archi.chitose-d.net`を入力し、HTTPS強制設定を有効化。
    - ドメイン管理サービス（さくらインターネット）のDNS設定画面で、`archi`というエントリー名の`CNAME`レコードを`yukiko-archi.github.io.`に設定（末尾のドットを含む）。
- **現状**: さくらインターネットでのDNS設定は完了し、DNS伝播が完了してカスタムドメイン (archi.chitose-d.net) でのアクセスを確認済み。

### 4. お問い合わせ方法の決定
